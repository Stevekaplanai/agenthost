// how it works

The migration is the product.

Anyone can start a cloud container. The hard part is your agent arriving with its memory intact. The transcript up top is a real run, first try, numbers untouched.

[1/5]
Detects your harness. Claude Code from ~/.claude, Hermes from its home. Skills, agents, memory, config.
[2/5]
Packs it, minus your secrets. Credential files never leave disk. Keys inside configs get redacted and re-provided as encrypted secrets.
[3/5]
Checks cloud compatibility. MCP servers pointing at localhost get flagged and disabled, not silently broken.
[4/5]
Deploys to YOUR Fly.io account. App, encrypted volume, secrets, container. Repos clone fresh with a scoped token.
[5/5]
Hands you a link. Tap it on your phone: live terminal, cookie login, no passwords to type. It survives lid-close, sleep, and airport wifi.
// security

There is no server to trust.

your laptop → flyctl (on your machine)
  → your Fly.io account (encrypted secret store)
  → your container

✗ AgentHost servers: none exist
✗ telemetry on secret values: none
✗ credential files migrated: never
  • Auth is a value you hand over: a claude setup-token (default) or your own API key. Never lifted from disk.
  • GitHub access is a fine-grained token scoped to the repos you pick. Never account-wide.
  • Every .env entry needs your per-file consent before it becomes a cloud secret.
  • MIT licensed. Read exactly what leaves your machine.
  • One command teardown: agenthost destroy removes app, volume, and secrets.

// questions

Asked and answered.

Is this safe? Where do my keys go?
Your laptop hands them to flyctl. flyctl puts them in your own Fly.io account's encrypted secret store. Your container reads them there. There is no AgentHost backend to breach. The CLI has zero telemetry on secret values, and --dry-run shows the exact manifest before anything moves.
Is this against Anthropic's terms?
The cloud agent authenticates with a value you explicitly provide. That means a claude setup-token (the supported headless path for subscribers) or your own API key. AgentHost never lifts credential files or login sessions off your disk.
My machine is always on. Why would I need this?
Then you are the parallel-agents customer. Run the deploy twice and you have two agents. One builds. One watches GitHub. Neither steals CPU while you render video. Your machine goes back to being yours.
What does it cost to run?
$11 to $12 per month for an always-on box with 2GB of RAM. Fly.io bills you directly. Claude usage is separate. The CLI itself is free.
Codex? OpenClaw? Other agents?
Claude Code is fully supported today, Hermes is in beta, Codex and OpenClaw are next. Leave an email and you'll hear when yours lands:
How do I kill it?
agenthost destroy. App, volume, secrets: gone from your account. One command in, one command out.
